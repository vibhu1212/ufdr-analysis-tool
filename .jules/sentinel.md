## 2024-05-24 - [Avoid `eval()` on external metadata]
**Vulnerability:** Found `eval()` being used in `media/video_processor.py` to parse video frame rates from FFmpeg probe output (`video_stream.get('r_frame_rate', '0/1')`). This is a critical Remote Code Execution (RCE) vulnerability because an attacker could craft a malicious video file with embedded Python code in its metadata, which would be executed when the system runs `ffmpeg.probe()`.
**Learning:** External data from tools like FFmpeg should be treated as untrusted input. The assumption that video metadata will always be safely formatted as `"num/den"` is insecure.
**Prevention:** Never use `eval()` to parse strings. Use safe parsing methods such as string splitting (`str.split()`), regex, and explicit type conversion (`float()`, `int()`).

## 2025-04-25 - Dynamic Table Name SQL Injection in PRAGMA and SELECT Statements
**Vulnerability:** Critical SQL Injection vulnerability found in `frontend/components/ufdr_upload_component.py` (during UFDR ingestion) and `rag/report_generator.py`. Table names sourced dynamically (e.g., from an uploaded SQLite database or predefined lists) were concatenated directly into formatted strings (`f"SELECT * FROM {table}"`, `f"PRAGMA table_info({table})"`) without sanitization or quoting.
**Learning:** SQLite does not allow parameterization (using `?`) for table or column names, only for values. Because untrusted `.db` files from user uploads can contain malicious table names (e.g., `sqlite_master; DROP TABLE cases;`), injecting them directly into queries can lead to full database compromise or unauthorized schema modifications.
**Prevention:** When dealing with dynamic table names, always validate the name using Python's `.isidentifier()` method to ensure it only contains valid characters (alphanumeric and underscores) and does not contain spaces or SQL symbols. Additionally, always properly double-quote the table name within the formatted string (e.g., `f'SELECT * FROM "{table}"'`).
